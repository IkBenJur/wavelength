# Workflow Rules

## Branching
- Never work directly on `master`. Before starting any feature work, create and switch to a new branch with a descriptive name (e.g. `feature/my-feature`).

## Before Committing
- Before committing any changes, start the dev server and explicitly ask the user to test the feature.
- Wait for the user's approval before proceeding with the commit.

## Pull Requests
- When a feature is complete and committed, open a PR to merge the feature branch into `master`.
